For developers
API
The application only counts with one single API route:
http://app/sendemail

GET /sendemail
Returns JSON with a single key: status. 

{ "status" : "up" }
The purpose of this route and method is to check if the service is up.

POST /sendemail

This route can be used to send the emails.
Route needs at least 3 parameters but not obligatory:
JSON with following keys: 

{
    "subject":"Subject of the Email",
    "text":"Plaintext content of the Email.",
    "html":"HTML content of the Email."
}
If the process is successful, the following JSON is returned: 

{ "status": 'success', "description": 'email sent' }
If the process is failed, the following JSON is returned: 

{ "status": 'failed', "description": 'error 500' }

SendGrid

The application works with SendGrid service. If you want to run your own NodeJS Email Sender you must use SendGrid service or to modify the code.
