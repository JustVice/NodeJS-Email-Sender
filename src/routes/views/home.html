For developers
API
The application only counts with one single route for API:
GET & post on http://app/sendemail

GET /sendemail
Returns JSON with a single key: status.
Example: {"status":"up"}
The purpose of this route and method is to check if the service is up.

POST /sendemail

This route can be used to send the Emails.
Route needs at least 3 parameters but not obligatory:
JSON with following keys: 

            {
                "subject":"Subject of the Email",
                "text":"Plaintext content of the Email.",
                "html":"HTML content of the Email."
            }
If the process is successful, JSON with key status will be success.

SendGrid

The application works with SendGrid service. If you want to run your own Email Sender with this software you must use SendGrid service or to modify the code.

Notice

If you try this service, even though you can see a success message when attempting to send an Email, it probably wasn't send.
SendGrid only allows verified accounts through it's internal settings to send Emails.
